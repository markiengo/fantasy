# Data Mapping Files

This directory contains mapping files that translate external IDs (from ESPN API) to internal database IDs.

## Files

### idmap.json
- **Purpose**: Maps ESPN player IDs to internal database `player_id` values
- **Structure**: `{ "espn_player_id": internal_player_id }`
- **Usage**: Used by `tools/repeat/load_stats.py` and `POST /api/load-stats` to translate ESPN API player responses to database IDs
- **Generated by**: `seed_players.py` during initial data seeding
- **Example**: `"225561": 18` means ESPN player ID 225561 corresponds to database player_id 18

### matchmap.json
- **Purpose**: Maps ESPN event/match IDs to internal database `match_id` values
- **Structure**: `{ "espn_event_id": internal_match_id }`
- **Usage**: Used by `tools/repeat/load_stats.py` and `POST /api/load-stats` to translate ESPN API match responses to database IDs
- **Generated by**: `seed_team_matches.py` during initial data seeding
- **Example**: `"760415": 1` means ESPN event ID 760415 corresponds to database match_id 1

## Workflow

1. Run `seed_team_matches.py` → generates `matchmap.json`
2. Run `seed_players.py` → generates `idmap.json`
3. Run `tools/repeat/load_stats.py` or call `POST /api/load-stats` → uses both maps to load ESPN data into the database through the scoring path

## Notes

- These files are static once generated during initial seeding
- They enable the ESPN API integration to work with your internal database schema
- If you add new players or matches manually, you may need to update these mappings accordingly
